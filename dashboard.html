USDTNGN — buy/sell slippage vs mid
Buy slippage % Sell slippage % G1 partial-fill
Buy slip % = ((weighted avg buy price ÷ mid) − 1) × 100. Sell slip % = ((weighted avg sell price ÷ mid) − 1) × 100. Each point walks the visible orderbook cumulatively until 100k USDT is consumed, clipping the boundary layer. Mid is itself a small depth walk — the average of 1k USDT walked on each side, not raw best_ask/best_bid — so a lone dust order at the touch can't distort the reference. Points inside the last hour are 5-second snapshots; older points are hourly averages.
USDTNGN — liquidity uptime (trading hours 9am–6pm NGT)
Ask uptime % Bid uptime %
The graph is the persisted series: a ±0.071% band around live mid — that percent is 1₦ ÷ 1400₦ target price, applied multiplicatively so the band tracks mid. Per 5-second poll, a side scores 1 if at least 100k USDT of resting size sits within its band, else 0. Each hourly point is the fraction of that hour's usable polls that passed, per side. The highlighted point is the current in-progress hour — the same value shown live in the ±0.071% stat cards above, so you don't have to wait for the hour to close. A poll where the book was empty on a side produces no sample and doesn't count either way.

Pricing & circuit breaker
B1 — extra tolerance beyond each pair's own expected offset (target_spread/2) from the MEXC/KuCoin reference. The bot doesn't quote symmetrically on the reference, so some offset is normal; this is the EXTRA room allowed on top of that before B1 fires. USDT-quoted pairs only.
B2 — % diff between MEXC and KuCoin themselves before one is flagged an outlier.
B3 — consecutive failed-to-resolve reads before a dead feed fires (Tier 2). API error / unlisted / wrong alias.
B3 — consecutive unchanged reads before the cross-source liveness check runs. Fires Telegram only if the peer source is still moving; otherwise (calm market / single-source) it's dashboard-only.
B3 — a move ≤ this % counts as "unchanged"; the same threshold decides whether a peer is "moving". 0 = exact equality (original behaviour). Raise slightly to absorb sub-tick wiggle.
B4 — total price move % within the k-line window treated as breaker-risk territory.
B4 — fires HIGH at this fraction of the breaker %, CRITICAL at 100%+.
F1 — % gap between a pair's quoted price and its implied cross-rate before flagging.

Depth-walk slippage (G1) — USDTNGN only
Base-asset quantity walked cumulatively down each side of the book. The boundary layer is clipped to only the remainder needed. Shared by buy and sell walks.
Mid price is the average of two small walks (this weight on each side), not raw best_ask/best_bid — a lone dust order at the touch can't distort the reference. Falls back to top-of-book if either side can't supply this weight; that sample is flagged.
How often the depth-walk task fetches USDTNGN and records a sample. Independent of the main cycle sleep — this loop runs on its own.
How long one raw 5s-sample bucket stays open before being condensed to a single hourly-average point. Default 3600 = one hour per bucket.
How long the hourly averages are retained on disk before being pruned. The chart date-picker archive can go back this far.
Liquidity uptime — USDTNGN only
Active band anchor. The band half-width is a percent p = 1₦ ÷ s × 100, applied to live mid as mid×(1±p/100) — so it equals ±1₦ only when mid = s and scales with mid otherwise. Lower s ⇒ wider band. Values ≤ 0 fall back to the config default s.
A side scores 1 for a poll when at least this much resting size sits within the ±p% band, else 0. Independent of the G1 walk weight above. Hourly uptime is the fraction of that hour's usable polls that passed, per side.
Alerts
How long a pair's Telegram alerts stay muted when you tap Suspend beside it below. One global value — every pair's button uses it. The pair keeps being monitored and stays on the dashboard; only its Telegram delivery is paused for the window (F1 alerts on other pairs are unaffected). Save the config first if you change this — the button uses the saved value.
